Asynchronous JavaScript

Asynchronous JavaScript allows your program to run tasks without blocking other operations, making your applications faster and more efficient.

What You Will Learn

  • What asynchronous programming is
  • How to use Promises
  • How to use async/await

Synchronous vs Asynchronous

// Synchronous
console.log("Start");
console.log("End");

// Asynchronous
console.log("Start");
setTimeout(() => {
  console.log("Delayed");
}, 1000);
console.log("End");

Async code allows tasks to run in the background without stopping the program.

Promises

let promise = new Promise((resolve, reject) => {
  let success = true;

  if (success) {
    resolve("Success!");
  } else {
    reject("Error!");
  }
});

promise
  .then(result => console.log(result))
  .catch(error => console.log(error));

Promises handle operations that complete in the future.

Async / Await

async function getData() {
  let response = await fetch("https://jsonplaceholder.typicode.com/posts/1");
  let data = await response.json();
  console.log(data);
}

getData();

async/await makes asynchronous code easier to read and write.

Error Handling

async function loadData() {
  try {
    let res = await fetch("invalid-url");
    let data = await res.json();
  } catch (error) {
    console.log("Error:", error);
  }
}

Use try...catch to handle errors in async code.

Best Practice

  • Use async/await for readability
  • Always handle errors
  • Avoid deeply nested callbacks

Try It Yourself

  • Create a promise that resolves after 2 seconds
  • Fetch data from a public API
  • Handle errors using try...catch
