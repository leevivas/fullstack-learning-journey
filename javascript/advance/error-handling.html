Error Handling

Error handling helps you detect, manage, and fix problems in your code. It ensures your application runs smoothly even when something goes wrong.

What You Will Learn

  • What errors are
  • How to use try...catch
  • How to debug your code

Types of Errors

  • Syntax Errors — mistakes in code structure
  • Runtime Errors — occur while the program runs
  • Logical Errors — incorrect results

Try...Catch

try {
  let x = y; // y is not defined
} catch (error) {
  console.log("Error:", error.message);
}

The try block runs code, and catch handles errors.

Finally

try {
  console.log("Trying...");
} catch (error) {
  console.log(error);
} finally {
  console.log("Always runs");
}

The finally block always runs, whether an error occurs or not.

Throw Custom Errors

function checkAge(age) {
  if (age < 18) {
    throw new Error("Must be 18 or older");
  }
}

try {
  checkAge(15);
} catch (error) {
  console.log(error.message);
}

You can create your own errors using throw.

Debugging Tips

  • Use console.log() to check values
  • Read error messages carefully
  • Use browser developer tools

Best Practice

  • Always handle possible errors
  • Write clear error messages
  • Don’t ignore errors

Try It Yourself

  • Create a try...catch block
  • Throw a custom error
  • Debug a simple mistake
