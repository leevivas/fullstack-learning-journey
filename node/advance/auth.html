Authentication

Authentication ensures that users can securely log in and access protected parts of your application.

What is Authentication?

Authentication is the process of verifying a user's identity. It ensures only authorized users can access certain parts of your app.

Basic Authentication Flow

  • User enters email and password
  • Server verifies credentials
  • Server returns a token or session
  • User accesses protected routes

Password Hashing

npm install bcrypt
      
const bcrypt = require("bcrypt");

const hashedPassword = await bcrypt.hash("mypassword", 10);

Never store plain passwords. Always hash them.

Verify Password

const match = await bcrypt.compare("mypassword", hashedPassword);

Compare user input with stored hash.

JSON Web Token (JWT)

npm install jsonwebtoken
      
const jwt = require("jsonwebtoken");

const token = jwt.sign({ userId: 1 }, "secretkey");

JWT is used to authenticate users after login.

Protect Routes

function authMiddleware(req, res, next) {
  const token = req.headers.authorization;

  if (!token) return res.send("Access denied");

  next();
}

Middleware protects routes from unauthorized access.

Why Authentication Matters

  • Protect user data
  • Secure applications
  • Enable login systems

Best Practice

  • Hash passwords (bcrypt)
  • Use JWT or sessions
  • Never expose sensitive data
