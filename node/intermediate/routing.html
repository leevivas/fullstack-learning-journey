Routing

Routing allows your application to respond to different URLs (endpoints). It helps organize how your server handles requests.

What You Will Learn

  • What routes are
  • How to handle multiple endpoints
  • How to organize routes in Express

What is a Route?

A route defines how your server responds to a specific request URL. 

app.get("/", (req, res) => {
  res.send("Home Page");
});

Multiple Routes

app.get("/about", (req, res) => {
  res.send("About Page");
});

app.get("/contact", (req, res) => {
  res.send("Contact Page");
});

Each route handles a different endpoint.

Route Parameters

app.get("/user/:id", (req, res) => {
  res.send("User ID: " + req.params.id);
});

Route parameters allow dynamic values in URLs.

Query Parameters

app.get("/search", (req, res) => {
  res.send("Query: " + req.query.q);
});

Access query values from the URL.

Organizing Routes (Router)

// routes/user.js
const express = require("express");
const router = express.Router();

router.get("/", (req, res) => {
  res.send("User List");
});

module.exports = router;
      
// app.js
const userRoutes = require("./routes/user");

app.use("/users", userRoutes);

Use routers to keep your code clean and modular.

Why Routing Matters

  • Organizes backend logic
  • Handles multiple endpoints
  • Improves scalability

Best Practice

  • Group related routes
  • Use routers for large apps
  • Keep routes simple and readable

Try It Yourself

  • Create /about and /contact routes
  • Create a route with parameters
  • Organize routes into a separate file
