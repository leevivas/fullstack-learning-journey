Semantic HTML

Semantic HTML uses meaningful tags to describe the structure of a webpage. It makes your code easier to read, maintain, and understand.

What You Will Learn

  • What semantic HTML is
  • Common semantic elements
  • Why semantic structure is important

What is Semantic HTML?

Semantic HTML uses tags that describe their meaning. Instead of using generic elements like <div>, you use elements like <header>, <section>, and <footer>.

Example

<header>
  <h1>My Website</h1>
</header>

<main>
  <section>
    <h2>About</h2>
    <p>This is a sample page.</p>
  </section>
</main>

<footer>
  <p>Copyright 2026</p>
</footer>

Common Semantic Tags

  • <header> — top section of a page
  • <nav> — navigation links
  • <main> — main content
  • <section> — grouped content
  • <article> — independent content
  • <footer> — bottom section

Why It Matters

  • Improves readability of your code
  • Helps search engines understand your site (SEO)
  • Improves accessibility

Try It Yourself

  • Create a page using header, main, and footer
  • Add a section with a heading and paragraph
  • Avoid using only <div> tags
